# Target Contract Migration

This note tracks shipped-vs-target gaps while `DESIGN.md` stays clean as the
target contract.

Current state: the spec-layer spine is executable and validated across multiple
concept shapes, but some names and envelopes still carry migration-era vocabulary.

| Area | Shipped today | Target contract | Migration status |
| --- | --- | --- | --- |
| Output constructors | `bin_output()`, `num_output()`, `cat_output()`, `struct_output()` are the only surface; the legacy `binary_output()`/`number_output()`/`categorical_output()`/`fields_output()` aliases are removed. | `bin_output()`, `num_output()`, `cat_output()`, `struct_output()` | Completed at the constructor surface. Internal `$kind` (binary/number/categorical/fields) is unchanged; rename only if the runner surface needs it. |
| Absence policy | Removed from `variable_spec()` and templates; old `absence_policy=` calls now fail loudly. | Coverage/audit semantics owned by output and channel evaluability | Completed for the spec layer; preserve uncertainty through `channel_coverage`, `channel_status`, and evidence/audit views. |
| Source registry and roles | Default source specs carry redsan-shaped metadata + canonical role names; the code/act executor resolves its columns from roles (`.code_source_binding()`), so two coded sources with different physical code columns (`pmsi$diag`→`diag`, `pmsi$actes`→`CODEACTE`) share one executor. The dead `legacy_roles` migration-alias apparatus is removed. Normalized output columns keep physical/runner names. | `source_spec()` is seeded from `redsan::edsan_sources()` and exposes an open canonical role set: identifier spine, point/interval time roles, query/batch date keys, payload roles, and auxiliary row coordinates. | Executors-consume-roles is done on the code/act path; output frames stay physical by design (executors resolve names from roles, they don't require renamed frames). Deferred (no consumer): (1) `.lab_source_binding()` — the lab executor still names biology columns directly, but biology has a single source, so there is no second consumer forcing role-resolution (contrast the code path's two sources); (2) `redsan::edsan_sources()` seeding — specs are hand-declared. Add each when a real second-warehouse/second-biology-source consumer exists. |
| Channel shape | `produces` is derived from the channel `type` inside the constructor (never user-supplied); `act_channel()` exists and shares the code-membership executor. | Constructor implies signal shape; `act_channel()` for CCAM | Completed. `produces` is internal-only (read at assembly, never written by callers); the CCAM act route ships with `act_channel()` + `ccam()` over `pmsi$actes`. |
| Text method | `llm_after_lucene()` takes no candidate knob; the dead `top_n` (set-but-not-read, text is pre-retrieved) has been removed. | `llm_after_lucene(candidates = llm_candidate_selection(arrange=..., limit=...), prompt=..., type=..., positive_hit_when=...)` | Selection helper named `llm_candidate_selection()` (disambiguated from the structured `candidates` frame). Deferred until retrieval runs in-engine and there is a consumer; the name is reserved, not built. |
| Boolean envelope | `values` is `value` + `channel_coverage`; `channel_status`/`membership`/`evidence` carry no `role`; `overlap` carries the pattern-determined `value` + `channel_coverage`. | Public output centered on `value`, `channel_coverage`, membership/audit, evidence, `combine_rule` | Completed. `decision`/`decision_state` removed (observed set algebra is always determined; included/excluded is a presentation recoding of `value`). Public `role`/polarity removed everywhere; expression polarity is derivable internally when a sugar-lowering consumer needs it (the orphaned `.hitset_expr_roles()` was deleted with its only consumer). |
| Inspection | Minimal `inspect()` / `resolve_variable_spec()` exists for concept, channel, and variable specs. | `inspect()` and `resolve_variable_spec()` expose inherited channel defaults and final executable view | Keep as experimental; broaden only as later slices reveal missing fields. |
| Event/stay grain | Text event linkage exists; structured event/stay linkage incomplete | Event/stay grain works across relevant channel types | Add structured event/stay linkage as an executor extension. |
| Whole-history text | Whole-history structured code AND text both execute with no anchor/window. `.retrieve_text_channel` scopes subject-linkage-with-no-window to the subject's ENTIRE document record (no date filter), mirroring the whole-history code path; whole-history tasks carry no anchor_date. | Whole-history subject text can retrieve all subject documents | Completed. No-window subject text eligibility is wired generically. Validated through `run_variable()` by a disposable whole-history text variable_spec (`window = NULL` over an existing text channel) in the whole-history slice test — a test-local probe, not shipped concept machinery. |
| LLM boundary | Ellmer owns the structured call (`chat$chat_structured`) and type validation/parsing (`ellmer::type_from_schema()`); the engine owns everything else through the per-task-isolated `run_extraction` loop: candidate selection (pre-retrieved), prompt rendering (`prompt_builder`), evidence-ID validation (`resolve_cited_ids()` + `.materialize_task_evidence()`), response-to-hit mapping (parser `normalized_value` → observed hit), and provenance (`attempts` audit with model/seed/prompt+schema+query hashes/finish-reason/raw response, plus evidence + `channel_status`). Method-specific knobs live in the `definition`/`extractor` bundle; `llm_after_lucene()` is a declarative tag that `inspect()` surfaces but dispatch never reads (the text arm keys on `channel_def$type` and consumes `extractor`). | Ellmer owns structured call/type validation/parsing; engine owns candidate selection, prompt rendering, evidence-ID validation, response-to-hit mapping, provenance | The who-owns-what boundary is in place and shipped; both sides are already implemented. The only unaligned part is the DECLARATIVE surface (DESIGN §488): fold the method-specific knobs (`prompt`, `type`, `candidates`, `positive_hit_when`) INTO the `llm_after_lucene(...)` signature instead of the bundle. This is the SAME reserved-not-built work as the Text-method row — deferred until in-engine retrieval and a consumer exist. `positive_hit_when` is likewise reserved: response-to-hit mapping is currently the binary parser's `documented`→`present`→hit; add the literal when a variable needs a mapping the parser does not already bake in. |

Tests should distinguish contract tests from temporary migration tests. Before
freezing a public API rule, exercise it against code, text, thresholded lab,
unthresholded lab, patient-grain, and event/stay-grain variables.
